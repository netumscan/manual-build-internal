传输前导字符#

前导字符（国家代码和系统字符）可以作为 UPC-A 符号体系的一部分进行传输。选择以下选项之一，设置并向主机传输 UPC-A 前导字符：仅传输系统字符、传输系统字符和国家代码（美国为“0”）或不传输前导字符。

传输前导字符#

前导字符（国家代码和系统字符）可以作为 UPC-A 符号体系的一部分进行传输。选择以下选项之一，设置并向主机传输 UPC-A 前导字符：仅传输系统字符、传输系统字符和国家代码（美国为“0”）或不传输前导字符。

传输校验字符#

将 UPC-E 转换为 UPC-A#

选择“启用”后，将 UPC-E 解码数据转换为 UPC-A 格式。转换后的数据将遵循 UPC-A 格式，并受 UPC-A 编程选项（例如，前导码、校验位）的影响。

启用/禁用#

EAN-8 扩展#

当选择“启用”时，扫描仪会在解码的 EAN-8 条码中添加五个前导零，以扩展至 13 位数字。

UPC/EAN/JAN 附加码#

备注

  • 禁用：当扫描枪扫描带有附加码的条码时，它会解码该条码并忽略附加码。扫描枪也可以解码不带附加码的条码。

  • 启用：扫描仪仅解码带有附加代码的混合条形码。

  • 自适应：扫描仪可以解码带有或不带有附加代码的混合条形码。

将 Code 39 转换为 Code 32（意大利药品代码）#

Code 32 是意大利制药行业使用的 Code 39 的变体。扫描下方相应的条形码即可启用或禁用 Code 39 到 Code 32 的转换。

Code 32 前缀#

启用此参数会将前缀字符“A”添加到所有 Code 32 码。启用此参数之前，您必须将 Code 39 转换为 Code 32（意大利药品代码）。

传输校验字符#

校验位验证#

此功能允许扫描器检查所有 Code 11 码制的完整性，以验证数据是否符合指定的校验位算法。这将选择解码后的 Code 11 条码的校验位机制。选项包括检查一位校验位、检查两位校验位或禁用该功能。
